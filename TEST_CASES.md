## 案例 1: 高物理伤害弓 — 点伤 + 物理% + 攻速

**输入 (CN 模式):**
```
lang: 'cn', classIds: ['bows'],
modIds: ['flat_phys', 'physDmg', 'attackSpeed'],
hasQuality: true, ilvlMin: 65
```

**生成 Regex (短版):**
```
"弓|附加 \d+ 至 \d+ 物理伤害|物理伤害提高 \d+%|攻击速度提高 \d+%|品质.*\+\d"
```
长度: 54字符 → **短版压缩**:
```
"弓|附加 \d+ 至 \d+ 物理伤害|物理伤害提高 \d+%|攻击速度提高 \d+%"
```
长度: 43字符 ✅

**匹配的中文游戏文本:**
- "弓" — 武器基底
- "附加 15 至 30 物理伤害" — 物理点伤
- "物理伤害提高 85%" — 物理%
- "攻击速度提高 15%" — 攻速

**为什么有效:** 点伤格式在 PoE2 中文客户端为 "附加 X 至 Y 物理伤害"，`\d+` 通配任意数字

## 案例 6: 引路石 — 好前缀 + 坏后缀全部

**输入 (CN 模式):**
```
lang: 'cn',
mapModIds: ['ws_rarity','ws_quantity','ws_pack_size','ws_ref_phys','ws_ref_ele','ws_no_regen','ws_pen']
```

**生成 Regex:**
```
"物品稀有度提高|物品数量提高|怪物群规模提高|怪物.*反射.*物理伤害|怪物.*反射.*元素伤害|无法.*回复|不能.*回复|穿透.*元素抗性|穿透.*抗性"
```
长度: 90字符 → **极限短版:**
```
"物品稀有度提高|物品数量提高|怪物群规模提高|怪物.*反射.*物理伤害|无法.*回复|穿透.*元素抗性"
```
长度: 63字符 → **再压缩:**
```
"物品稀有度提高|物品数量提高|怪物.*反射|无法.*回复|穿透"
```
长度: 38字符 ✅

**匹配的中文游戏文本:**
- "物品稀有度提高 80%" — 稀有度 (好)
- "物品数量提高 25%" — 数量 (好)
- "怪物反射 15% 物理伤害" — 反射 (坏)
- "玩家无法回复生命和魔力" — 无法回复 (坏)
- "怪物伤害穿透 10% 元素抗性" — 穿透 (坏)
